# Easyslim - 智能减肥追踪工具

Easyslim 是一款高颜值、离线优先（Offline-First）的智能体重与饮食健康追踪应用。它专为希望通过多元健康菜系（支持国风中餐、美式轻卡、和风日式等 4 大子系列食谱）和精准卡路里管理实现科学减重的人群设计。

本工具不仅适配 PC 桌面端，更深度适配了 iOS/Android 等移动端设备，支持以渐进式 Web 应用（PWA）模式添加到手机桌面独立运行，享有秒开、离线查看和纯净全屏的使用体验。

## 🌟 核心功能亮点

### 1. 双时间体重波动追踪与自动差值评估
*   **清晨空腹体重 (🌅 Morning Weight)**：每天清晨起床排便后测得的净重，作为评估长期燃脂趋势的核心指标。
*   **睡前放松体重 (🌙 Bedtime Weight)**：每天临睡前的体重，用于监测白天的水分潴留和进食摄入。
*   **早晚体重差自动分析**：
    *   差值在 `0.5 ~ 1.0 kg`：代表机体代谢效率处于非常健康的活跃状态。
    *   差值 `> 1.2 kg`：系统将发出警示，提示晚餐进食可能偏油腻、过咸，导致水分潴留。
    *   差值 `< 0 kg`：提示可能由于白天喝水不足或能量赤字偏大。

### 2. 多渠道文本 AI 饮食解析器 (免 Key 直连 & 自备 Key)
Easyslim 接入了先进的大语言模型接口，支持使用自然语言（如：“我早上吃了一个水煮蛋、两片全麦面包、喝了一盒纯牛奶”）打卡。
*   **内置免配置通道 (基于 Puter AI)**：新用户无需注册任何 API Key 或进行繁琐配置，即可直接免费使用。首次点击时会弹出浏览器授权提示，一键点击 Allow 即可立即体验高准确率的 GPT-4o-mini 食物解析。
*   **自备 API Key 通道**：全面支持自备 Google Gemini、DeepSeek、硅基流动 (SiliconFlow)、自定义 OpenAI 兼容代理等服务。
*   **高度自由的端点配置**：允许自定义 **API 接口地址 (Base URL)** 与 **模型名称 (Model)**，默认已为您预填对应提供商的最佳推荐配置。
*   **高颜值 step-by-step 连通性测试模态框**：在配置中输入参数后，可触发连接测试。测试框会通过平滑微动画展示三步校验流程（验证参数、API 握手连接、解析格式验证），如果连接失败会详细给出诊断代码、状态码及解决方案。

### 3. 多种科学断食/减重模式与进食窗口深度配置
*   **科学亏空与安全底线**：根据您输入的性别、年龄、身高、当前体重及运动活跃度，利用 **Mifflin-St Jeor** 公式自动算出您每日的基础代谢率 (BMR) 与总消耗量 (TDEE)，并自动将亏空热量安全控制在合理下限（不低于 BMR 的 90%）。
*   **多样化断食减重模式 (Fasting Patterns)**：
    *   *标准一日三餐* (Standard 3-Meals)：最稳妥的健康减脂，常规推荐早、午、晚餐。
    *   *16+8 间歇性断食* (16:8 Fasting)：将一天中所有的食物摄入限制在 8 小时内，断食 16 小时。可深度配置进食窗口起始时间，并以高颜值 24 小时时间盘展示。
    *   *20+4 战士断食* (20:4 Fasting)：将一天食物限制在 4 小时内，断食 20 小时。
    *   *5+2 轻断食模式* (5:2 Fasting)：每周自由选择 2 天进行极低热量摄入（约 500-600 大卡），另外 5 天正常推荐。
*   **阶段减重里程碑**：根据设置的达标时长，将目标合理拆解为均等的月度里程碑，并自动检索历史体重计算是否达成。

### 4. 多元健康菜系选择与 4 大减脂子系列
为迎合不同的饮食文化与喜好，Easyslim 首批落地了 3 大菜系，且每个菜系均支持 4 个健康的控卡子系列：
*   **🇨🇳 国风健康中餐 (Chinese Cuisine)**：遵循《中国居民膳食指南》餐盘模型与 CCDC 营养成分库，主打少油快炒、清蒸与温拌。
*   **🇺🇸 美式轻卡膳食 (American Light)**：遵循美国 USDA MyPlate 膳食指南，以烤箱烘烤、无油肉饼和高蛋白轻沙拉为特色。
*   **🇯🇵 和风日式料理 (Japanese Cuisine)**：遵循 WHO 西太平洋健康饮食指南，利用日式昆布出汁高汤、凉拌和盐烤。
*   **4 大子系列食谱库**：
    *   *水油焖菜系列*：利用清水加少量食用油进行焖煮，最大限度保留蔬菜的营养成分和水分，饱腹感强。
    *   *轻食沙拉系列*：清爽低卡，搭配轻量油醋汁或希腊酸奶代酱，促进肠道代谢。
    *   *低碳生酮系列*：极低碳水、适度健康脂肪、高蛋白质，快速推动身体进入燃脂状态。
    *   *地中海膳食系列*：富含不饱和脂肪（如橄榄油）、全谷物、深海鱼及豆类，呵护心脑血管健康。

### 5. 今日“临时换口味”机制与餐食克重自适应动态缩放
*   **今日临时换口味**：无需改变个人中心的全局偏好设定。您随时可在食谱推荐页顶部一键点击切换不同的菜系。这仅仅作为“今日的临时选择”，在跨天或重新载入时会自动恢复为您默认的菜系配置。
*   **餐食勾选与热量动态转移**：
    *   每日计划进食的餐分（早餐/午餐/晚餐）均可自由勾选。如果由于断食等原因少勾选了某餐，系统会自动将亏空热量转移并分配到其余勾选的餐份中，使得剩下餐食的克重自适应放大，从而达成每日摄入目标。
    *   如果在某一餐您实际吃下的食物偏多（超标）或偏少，系统在计算剩余餐食推荐量时，会自动扣减或增加剩余推荐餐食的分量和卡路里，维持全天能量平衡。

### 6. 智能补餐推荐 (Smart Recommendation)
*   若您今日的总卡路里摄入低于系统计算出的“安全摄入下限”（防止饿坏身体降低代谢），控制台和饮食记录页会瞬间计算并激活“智能补餐推荐”。
*   为您量身计算并给出三大补餐方向（高蛋白肉蛋、多维蔬果、优质油脂坚果）的精确克重、大卡说明及补餐逻辑。

### 7. Excel 风格计划总览月历与双曲线图表
*   **一表掌握历史**：包含日期、晨重、睡前重、运动内容、餐饮外食备注及累计减重的九大列月历总表。
*   **周均值与周变化率**：以 7 天为一个周期，智能合并右侧单元格，自动为您计算这 7 天晨重的周均值，以及对比上一周的周减少比例（翡翠绿字体标注下降率）。
*   **点击跳转补卡**：支持点击月历的任意一行，直接跳转至该日期补录或修改数据。
*   **SVG 晨晚双体重曲线**：图表直观展示过去 7 天起床与临睡前的体重走势，助您科学把握身体水肿与掉重节奏。

### 8. 本地虚拟多账号物理隔离 (100% 隐私安全)
*   支持任意创建本地账号与密码，无需联网注册，无任何验证，100% 离线使用。
*   不同账号的数据采用独立的前缀密钥存储在浏览器的 LocalStorage 中，互不干扰，完全物理隔断。
*   提供导出与导入数据备份的合并功能，支持无损地在多个设备之间进行数据合并。

## 🚀 部署与使用指南 (详细步骤)

### 💻 第一部分：在电脑 (PC) 上运行

1.  **准备环境**：您的电脑上需要安装有 [Node.js](https://nodejs.org/)。
2.  **克隆或下载项目**：
    将项目源码下载并解压到您的电脑本地，例如解压至 `D:\Agent\Weight Loss Tracking Tool`。
3.  **安装并运行轻量级 Web 服务器**：
    打开终端或 PowerShell，切换至项目根目录，运行以下命令：
    ```bash
    npm run dev
    ```
    *（注：此命令内部实际上执行了 `npx -y http-server -p 8080 -c-1`。它将自动下载无缓存的轻量级静态服务包，并在 `8080` 端口启动开发服务器。）*
4.  **访问项目**：
    在浏览器中输入以下地址即可开始使用：
    ```text
    http://localhost:8080
    ```

---

### 📱 第二部分：在 iOS 手机端部署（添加至桌面以实现离线全屏）

要想在脱离电脑 Wi-Fi 的移动网络环境下、甚至在完全断网（离线）时也能正常打开和查看内容，请务必严格执行以下步骤将应用安装为 **iOS PWA**：

1.  **手机接入同一个局域网**：
    确保您的电脑和手机连接在同一个 Wi-Fi（局域网）下。
2.  **获取电脑的局域网 IP 地址**：
    *   在 Windows 电脑上：打开命令行（Cmd）或 PowerShell，输入 `ipconfig`，找到您的“无线局域网适配器 Wi-Fi”下的 `IPv4 地址`（例如 `192.168.1.100`）。
3.  **在手机 Safari 上打开**：
    *   打开 iOS 手机上的 **Safari 浏览器**（必须是 Safari，其他第三方浏览器如微信内置浏览器、Chrome iOS 版等均不支持 PWA 添加桌面功能）。
    *   在地址栏中输入：`http://[您的电脑局域网IP]:8080`（例如 `http://192.168.1.100:8080`）。
4.  **添加到主屏幕 (Add to Home Screen)**：
    *   在 Safari 浏览器底部工具栏上，点击 **“分享”** 按钮（带箭头的方框图标）。
    *   在弹出的分享菜单中向下滚动，找到并点击 **“添加到主屏幕” (Add to Home Screen)**。
    *   修改或保留名称，点击右上角的 **“添加”**。
5.  **离线与移动网络测试**：
    *   现在，手机桌面上会生成一个名为“轻盈减重”的应用图标。
    *   **断开电脑 Wi-Fi，切换至手机移动网络，或者直接开启飞行模式（断网）**。
    *   点击桌面的“轻盈减重”图标，您将看到它以纯净的全屏独立运行（不带有 Safari 的浏览器地址栏和底部导航栏）。
    *   得益于 `sw.js` (Service Worker) 的缓存控制，即使在断网状态下也能秒开，并且能正常查询、打卡记录、阅读推荐食谱！

### ⚡ 如何使用连接测试小框

1.  在表单中填入您选择的渠道、接口地址、模型名称和 API Key。
2.  点击 **“⚡ 测试 AI 连接”** 按钮。
3.  系统会弹出一个 **“🛠️ AI 连接测试”** 模态框，并依次展示三步检测进度：
    *   **第一步：验证 API 输入参数** — 检查必填项是否有遗漏。
    *   **第二步：与 API 服务器握手** — 真实向指定的 API 接口发送一条测试文本（`1个鸡蛋`）。
    *   **第三步：数据解析与格式验证** — 拦截服务器的返回，并尝试解析为标准食物 JSON。
4.  **测试成功**：第三步成功后，会显示翡翠绿色的成功框，并展示 AI 对 `1个鸡蛋` 的实际估算重量与热量结果，提示您可以保存并使用。
5.  **测试失败**：若其中某步出错，步骤栏会亮起红灯，并展现红色背景的**失败原因诊断分析**：
    *   *401 错误*：API Key 填写有误或已过期，请检查空格或复制完整度。
    *   *404 错误*：Base URL 填错。注意，Base URL 应为域名和基础路径，不应包含末尾的 `/chat/completions`。
    *   *400 错误*：填写的模型名称不被提供商支持，或该提供商不支持 JSON Schema 强制格式。
    *   *TypeError / Failed to Fetch*：跨域请求被浏览器拦截（CORS 限制），或在未开代理的情况下直连了不可访问的国外端点。建议配置国内中转代理。

## ⚠️ 重要注意事项与隐私安全说明

1.  **数据仅保存在当前设备本地**：
    Easyslim 没有中央后端数据库，所有的账户密码、身高体重档案、历史打卡数据都存储在您当前所用浏览器软件的 `LocalStorage`（本地缓存空间）中。
    *   **不要轻易清理浏览器缓存**，如果执行了“清理浏览器所有历史数据”或“深度清空网站 Cookie/存储”，您的打卡记录将会彻底丢失。
    *   建议定期在“数据分析”面板的最下方点击 **“📤 导出此设备备份”**，将数据保存为本地 `.json` 文件。
2.  **多端同步指南**：
    如果您在电脑上记录了数据，希望同步到手机上：
    *   在电脑上点击“数据分析” -> **“📤 导出此设备备份”**。
    *   将导出的 `.json` 文件通过微信、QQ、网盘等方式传到手机中。
    *   在手机上打开应用，进入“数据分析” -> **“📥 导入并合并备份”**，选择该 `.json` 文件。系统会自动去重并合并两端所有的打卡日志。
3.  **多账号安全与密码找回**：
    本应用的注册不需要任何身份验证（无需手机号/邮箱），属于离线物理隔断。
    *   **请务必牢记您的账号和密码**。一旦忘记，由于没有任何云端服务器支持，开发人员也无法为您找回密码。
    *   勾选 **“记住密码”** 会将账户信息保存在本地，方便后续一键登录。
4.  **AI 测试连接可能受浏览器 CORS（跨域资源共享）拦截影响**：
    由于是在浏览器客户端（前端）直接向第三方 AI 发送请求，如果第三方 AI 的网关没有开启 CORS 跨域许可，您的浏览器出于安全限制会拦截该请求并报 `TypeError: Failed to fetch`（网络连接失败）。
    *   *Puter.js*、*Gemini 官方端点* 和 *硅基流动 (SiliconFlow)* 均已在官方网关中开启了跨域许可，可直接在浏览器端安全调用。
    *   若使用自定义代理接口，请确保您的代理服务商已在 API 响应头中配置了 `Access-Control-Allow-Origin: *`。
